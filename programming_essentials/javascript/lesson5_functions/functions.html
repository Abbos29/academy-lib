Функции

Функции в Javascript

Что это такое?

Функция - это программный код, который определяется один раз и может вызываться на выполнение n-ое количество раз. Она состоит из последовательных инструкций - тело функции. Функция может принимать определенные значения и возвращать результат своего выполнения.

Проще говоря, если вам несколько раз в разных местах вашего кода нужно выполнить одну и ту же или похожую последовательность команд, вы можете вынести их в отдельную функцию, и вызывать ее, тем самым значительно уменьшив количество кода, которое нужно будет написать.

Как этим пользоваться?

Существуют три различных способов объявления функции:

  • Function declaration - функция объявляется с помощью ключевого слова function. 
function sayHello() {
  alert("Hello, Dan-it student!")
}

sayHello()
  • Function expression - функция также объявляется с помощью ключевого слова function, но у нее нет имени, и она записывется в переменную.
const sayHello = function() {
  alert("Hello, Dan-it student!")
}

sayHello()
  • Named Function expression - функция записывается в переменную, а также имеет свое имя (правда функцию нельзя будет вызвать по этому имени).
const sayHello = function danIt() {
  alert("Hello, Dan-it student!")
}

sayHello()

При этом, как видно, вызов функции идентичен для каждого случая:

sayHello()

Попробуем разобраться в чем разница:

Function declaration создается интерпретатором до начала выполнения Javascript кода, поэтому в коде ее можно вызывать до того места, где она была объявлена.

Например, следующий код будет успешно работать:

sayHello()

function sayHello() {
  alert("Hello, Dan-it student!")
}

Это происходит потому, что до начала выполнения всего Javascript кода, интерпретатор пробегается по файлу, находит все функции, объявленные как Function declaration, и запоминает их. И потом уже начинает выполнять сам код, позволяя обращаться к запомненным функциям.

Function expression и Named Function expression - функции, являющиеся частью какого-либо выражения (к примеру присваивания). Их можно вызывать только после объявления в коде.

Примечание
Функция, объявленная как "Named Function expression" имеет возможность обращаться к себе самой при рекурсивном вызове внутри самой функции.

Перенесем вызов нашей функции sayHello в начало кода, а также добавим console.log(sayHello)

Запустим в тренажере:

Function declaration

sayHello()
console.log(sayHello)

function sayHello() {
  alert("Hello, Dan-it student!")
}

Результат: функция запустилась, в консоли видим:

Function expression

Результат: ничего не произошло, так как фактически переменная уже объявлена, но операция присвоения функции идет после попыток ее вызвать и вывести в консоль.

Если выполнить тот же код в консоли браузера, вы увидите ошибку - так как происходит попытка обратиться к переменной до того, как она была объявлена:

Для Named Function expression результат будет аналогичен предыдущему.

Для чего это нужно на практике?

Параметры. При вызове функции ей можно передать данные, которые могут быть использованы в ходе выполнения программного кода (тела функции).

Например, на сайт зашел посетитель, ввел данные, и в зависимости от возраста нужно его пустить или не пустить к просмотру контента:

const validateUser = function(name, age) {
  if (age >= 18) {
    alert(`Welcome, ${name}`)
  } else {
    alert(`Sorry, ${name}, but you are too young`)
  }
}

validateUser('Andrew', 19)

Запустим этот код

Нашего Andrew пустили на сайт! Попробуйте поменять ему возраст и посмотрите на результат.

Параметров может быть бесконечное множество, и переданы они могут быть по-разному. Более детально об этом можно почитать здесь и здесь.

Возврат значения

Функция может вернуть результат своего выполнения, который будет использоваться в дальнейшем. Для возврата значения используется ключевое слово return.

Пример: 

function multiply(n, m) {
  return(n * m)
}

alert(multiply(5, 7))

Есть два числа, и нам необходимо вывести на экран их произведение. Запустите код.

return может находиться в любом месте функции. Как только до него доходит упраление - функция завершается и значение передается обратно в то место, откуда функция была вызвана.
Важно!
Внутри одной функции вызовов return может быть неограниченное количество.

Вернемся к примеру, в котором Andrew пытается попасть на сайт. Перепишем функцию validateUser используя return

const validateUser = function(age) {
  if (age >= 18) {
    return true
  } else {
    return false
  }
}

const name = 'Andrew'
const age = 18

if(validateUser(age)) {
  alert(`Welcome, ${name}`)
} else {
  alert(`Sorry, ${name}, but you are too young`)
}

В функции есть два слова return. Первое возвращает true, если условие выполняется успешно. Второе возвращает false.

Примечание
Помня информацию о приведении типов, функцию validateUser() из кода выше можно успростить следующим образом: 

const validateUser = function(age) {
  return age >= 18
}
Примечание
Если вы не напишете ключевое слово return в функции, оно все равно неявно будет добавлено интерпретатором Javascript в конце функции. При этом функция будет возвращать undefined. То же самое произойдет, если написать return, и не указать после него никакого параметра.

Попрактикуйтесь, изменяя параметр age.

Область видимости переменных

Рассмотрим простой пример (version1):

const a = 10

const sum = function(b) {
  return a + b
}

alert(sum(5))
console.log(`a === ${a}`)

Запустим этот код: получим 15. При этом обратите внимание на консоль: a === 10.

Чуть изменим пример (version2):

const a = 10

const sum = function(b) {
  const a = 7
  return a + b
}

alert(sum(5))
console.log(`a === ${a}`)

Запустим этот код: получим 12. При этом обратите внимание на консоль: a === 10.

Еще немного поиграемся с кодом (version3):

let a = 10

const sum = function(b) {
  a = 7
  return a + b
}

alert(sum(5))
console.log(`a === ${a}`)

Запустим этот код: получим 12. При этом обратите внимание на консоль: a === 7.

Разберемся что же тут произошло:

Все переменные внутри функции – это свойства специального внутреннего объекта LexicalEnvironment, который создаётся при запуске функции.

Этот объект является внутренним и он скрыт от прямого доступа извне.

В version1:

LexicalEnvironment = {b: 5}

При доступе к переменной интерпретатор сначала пытается найти переменную в текущем LexicalEnvironment, а затем, если её нет – ищет снаружи.

Именно поэтому переменная a в данном примере равна 10.

Важно!
Такой порядок поиска возможен благодаря тому, что ссылка на внешний объект переменных хранится в специальном внутреннем свойстве функции, которое называется [[Scope]]. Это свойство закрыто от прямого доступа, но знание о нём очень важно для понимания того, как работает JavaScript. При создании функция получает скрытое свойство [[Scope]], которое ссылается на лексическое окружение, в котором она была создана.

В version2:

LexicalEnvironment = {b: 5, a: 7}

Мы объявили переменную a внутри функции и она попала в ее лексическое окружение, поэтому внутри функции a === 7 и результат работы функции равен 12. Однако, в консоль выводится значение 10 - значение глобальной переменной a.

В version3:

LexicalEnvironment = {b: 5}

Мы переопределяем глобальную переменную a, поэтому результат работы функции равен 12. Так как мы внутри функции переопределили значение глобальной переменной a, в консоль выводится 7.

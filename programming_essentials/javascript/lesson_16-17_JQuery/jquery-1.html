Основы jQuery

Основы работы с jQuery

Что такое jQuery

При работе с DOM-деревом в JavaScript нам часто приходится совершать разные повторяющиеся однотипные действия, которые хотелось бы автоматизировать и вынести в отдельные функции, или еще лучше - методы универсального объекта. Например, мы хотим, чтобы при клике на любую кнопку с классом .btn-danger ей добавлялся класс active, и ее внешний вид менялся на противоположный (фон - красный, цвет текста - белый). Мы не знаем, сколько будет таких кнопок на странице, а значит должны написать универсальный код.

Как мы это делаем в базовом JavaScript? Допустим у нас есть следующая HTML страница.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>jQuery Test</title>
  <style>
    .btn {
      display: inline-block;
      padding: 10px 15px;
      text-align: center;
      text-decoration: none;
      border-radius: 5px;
    }
    .btn-danger {
      color: red;
      border: 1px solid red;              
    }
    .btn-danger.active {
      color: white;
      background-color: red;
    }        
    </style>
</head>
<body>
  <a href="#" class="btn btn-danger">Осторожно, милота!</a>
  <a href="#" class="btn btn-danger">Осторожно, JavaScript!</a>
  <a href="#" class="btn btn-danger">Осторожно, скидки!</a>
</body>
</html>

Чтобы выполнить требуемое действие, нам нужно будет написать следующий код:

const buttons = document.getElementsByClassName("btn-danger");

for(let i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener("click", toggleActive);
}

function toggleActive() {
  this.classList.toggle("active");
}

А если мы хотим сделать так, чтобы при клике на кнопку с классом .btn-danger класс active добавлялся ей, и убирался у всех остальных кнопок, то мы должно написать еще больше кода:

const buttons = document.getElementsByClassName("btn-danger");

for(let i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener("click", toggleActive);
}

function toggleActive() {
  const buttons = document.getElementsByClassName("btn-danger");

  for(let i = 0; i < buttons.length; i++) {
    buttons[i].classList.remove("active");
  }

  this.classList.toggle("active");
}

Нам приходится раз за разом в цикле обходить полученную коллекцию кнопок, чтобы повесить на них событие или удалить класс у всех. Поэтому рано или поздно возникает желание вынести все эти обработчики в отдельные функции и пользоваться ими постоянно, в разных проектах.

Поздравляю, вы решили создать свою библиотеку! А поскольку вы часто будете слышать этот термин в будущем, а также его сравнение с термином "фреймворк", поэтому давайте сразу внесем ясность - что это такое и в чем между ними разница.

Библиотека - это набор готовых решений, которые ускоряют разработку, позволяя не отвлекаться на написание рутинного шаблоного (повторяющегося) кода, и которые делают код короче и понятнее. При этом, что важно - библиотека не меняет саму логику построения программ. По сути вы делаете то же самое, только быстрее, проще и понятнее.

Фреймворк - это набор настолько высоокоуровненвых решений и заложеных в них идей (концепций), что для его правильного использования нужно менять значительную часть логику работы программы и ее построения. По сути, фреймворк диктует вам другой способ мышления при решении поставленной задачи. Примеры фреймворков - React, Vue, Angular.

К счастью, вам не нужно сейчас изобретать велосипед, вынося повтояющиеся части кода для работы с DOM'ом в отдельные функции - это уже сделал Джон Резиг в 2006 году, создав, наверное, самую знаменитую библиотеку JavaScript - jQuery, с которой мы и будем учиться работать.

jQuery - это библиоткка JavaScript, с помощью которой вы можете намного проще, удобнее и понятнее выполнять следующие операции:

  • находить нужные элементы в DOM-дереве;
  • манипулировать ими: добавлять, убирать и перемещать;
  • менять их CSS свойства и атрибуты;
  • вешать на элементы обработчики событий;
  • работать с элементами формы;
  • создавать красивую анимацию.

В результате вы:

  • пишете код быстрее;
  • он короче;
  • он понятнее для вас, и для тех, кто будет его читать;
  • его проще и удобнее менять.

Девиз jQuery - "write less, do more" (пиши меньше, делай больше).

Так как он нам все-таки может помочь? Скопируйте этот код с использованием jQuery себе в нижнюю часть HTML файла и убедитесь, что он делает то же самое, что и код выше на чистом JavaScript:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.3.1/jquery.min.js"></script>
<script>
  $(".btn-danger").click(function() { // вешаем обработчик события в виде анонимной функции на все элементы с классом btn-danger
    $(".btn-danger").removeClass("active"); // удаляем у всех элементов с этим классом клас active
    $(this).addClass("active"); // добавляем его текущему элементу
  });    
</script>

Как видите, кода стало намного меньше, и он стал интуитивно понятнее. Для этого и нужен jQuery. Давайте теперь научимся с ним работать!

Как подключить jQuery

Поскольку jQuery - это просто набор готовых методов, том нам нужно добавить его код на свою страницу, чтобы получить возможность его использовать. "Подключить" - означает просто добавить на страницу тег <script src="адресНужногоФайла.js"></script>, прописав в атрибуте src путь к нужному файлу (и неважно, где он находится). Это можно сделать двумя способами:

  1. Загрузка с удаленного сервера.

    Для этого просто напишите в секции head вашей страницы:

    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.3.1/jquery.min.js"></script>

    или 

    <script src="https://ajax.aspnetcdn.com/ajax/jQuery/jquery-3.3.1.min.js"></script>

    или 

    <script src="https://code.jquery.com/jquery-3.3.1.min.js" integrity="sha256-FgpCb/KJQlLNfOu91ta32o/NMZxltwRo8QtmkMRdAu8=" crossorigin="anonymous"></script>

    Актуальную ссылку на последнюю версию библиотеки всегда можно найти на сайте jQuery здесь.

  2. Скачать файл с кодом jQuery в папку js своего проекта и подключить его оттуда. Сделать это очень просто:

    • заходим на страницу https://code.jquery.com/;
    • Находим последний релиз, сейчас он под заголовком jQuery 3.x;
    • нажимаем правую кнопку мыши на ссылке с названием minified и выбираем пункт "Сохранить ссылку как";
    • сохраняем файл в папку js, которая находится в нашем проекте с нашей версткой, рядом с папками css, img и fonts внутри нашего проекта;
    • пишем в секции head <script src="js/jquery-номерВерсии.min.js"></script>. Сейчас, например, это будет <script src="js/jquery-3.3.1.min.js"></script>

Как использовать - нюансы синтаксиса

Синтаксис jQuery инструкций очень простой: $("селектор").метод();

Пример:

$(".btn").addClass("btn-danger");

Здесь:

  • $ - это то, с чего начинается любая инструкция на jQuery. По сути это - имя объекта, который содержит все нужные нам функции (как методы объекта) для работы с DOM;
  • сразу после, в круглых скобках и кавычках, идет селектор. Селектор - это CSS правило, по которому jQuery находит нужные элементы в DOM-дереве. Самое главное, что нужно о нем знать - здесь работают те же правила выбора элементов, что и в CSS-файле. В примере выше мы выбираем все теги с классом btn;
  • после селектора идет метод jQuery - встроенная в объект jQuery функция, которая делает что-то со всеми элементами, которые были выбраны по правилу, указаному в селекторе;

Обратите внимание на важные нюансы:

  • Правила выбора нужных элементов (селектор) нужно писать как в CSS, а не как в чистом JavaScript, через document.getElementsBy и children. Поэтому пишите селектор, как будто вы делаете это в CSS файле, не забывая про точки перед классами, правила наследования и вес CSS селекторов!
  • Методы jQuery всегда применяются ко всем выбранным элементам сразу, вам нет нужды прогонять их через цикл for, что значительно сокращает код и делает его понятнее.
  • Названия методов (addClass, css) более интуитивно понятны и отличаются от таких же в чистом JavaScript. А аргументы, передаваемые в методы, используют такие же CSS термины, как и файлах .css. Давайте разберем это на примере метода .css():

Этот метод добавляет выбраным элементам inline-стиль с css-свойством, которое передается в качестве аргументов. Первый аргумент - название свойства, второй - его значение. Как видите название свойства нужно указывать так же, как в css-файле. Он простой и с его помощью мы будем тренироваться дальше, пока не изучим новые методы. Давайте потренируемся:

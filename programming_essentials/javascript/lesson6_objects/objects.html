Объекты

Что такое объекты и зачем они нужны

Объект – это особый тип данных, который содержит внутри себя другие значения, которые можно менять, добавлять и убирать, а также функции для работы с этими значениями. Для чего нужны объекты:

  1. Хранить в одной переменной все данные о какой-то смысловой единице. Например, у вас есть несколько пользователей, и для каждого из них нужно хранить имя, фамилию и возраст. Если под каждое значение выделять отдельную переменную, то их получается очень много:

    const user1Name = "Алексей"; // имя первого пользователя
const user1LastName = "Серебряков"; // фамилия первого пользователя
const user1Age = 49; // возраст первого пользователя

const user2Name = "Андрей"; // имя второго пользователя
const user2LastName = "Данилко"; // фамилия второго пользователя
const user2Age = 42; // возраст второго пользователя

const user3Name = "Алина"; // имя третьего пользователя
const user3LastName = "Пьехова"; // фамилия третьего пользователя
const user3Age = 25; // возраст третьего пользователя

    В итоге получаются длинные имена переменных, в которых можно запутаться и забыть с какой-то из них произвести нужные операции. А если у пользователя будет не 3, а 10 характеристик, и пользователей будет не 3, а 50? Это делает программу на порядок сложнее и непонятнее. Хотелось бы, чтобы можно было просто объявить:

    const user1; // в этой переменной будут храниться все данные первого пользователя

const user2; // в этой переменной будут храниться все данные второго пользователя

const user3; // в этой переменной будут храниться все данные третьего пользователя

    И уже внутри этих переменных хранить все нужные данные - name, lastName, age и т.д.

  2. Хранить в одном месте не только данные, но и функции, которые будут работать с этими данными. Например, у вашего объекта user есть такие свойства, как имя, фамилия и отчество. Кроме того, нам для работы будут необходимы инициалы пользователя (ФИО). Но ФИО - не фиксированное значение, оно может измениться, если имя, фамилия или отчество изменятся. Значит, нам нужна функция, которая будет генерировать ФИО из первых букв фамилии, имени и отчества. Мы можем написать для этого отдельную функцию, но проблема в том, что при доработке программы названия переменных внутри объекта могут измениться, а значит функция станет нерабочей. И нам придется ее искать где-то в другом месте кода и менять в ней нужные значения. А если таких функций несколько? Удобнее, чтобы эта функция была уже встроена в сам объект - так проще менять ее код. Если нам нужны инициалы пользователя, мы просто пишем - user.showInitials(); - и все работает.

    Это очень удобно - когда все инструменты описаны в одном месте. Подробнее это будет рассмотрено в разделе Методы объекта.

Создание объекта со свойствами

Объект чаще всего создается так:

const имяОбъекта = {
  имяСвойства: "значение",
  "имяСвойства 2": "значение2",
  "имяСвойства 3": "значение3"
};

Свойствами объекта называются переменные, которые в нем хранятся.

имяСвойства - это имя переменной, которая хранится внутри объекта; значение - это данные, которые хранятся внутри переменной. Обычно это примитивный тип данных: строка, число, булевый тип (true/false), null, но может быть и другой объект.

Обратите внимание:

  • Для создания объекта нужно заключить его содержимое в фигурные {} скобки. Как только JavaScript видит такие фигурные скобки после = он понимает, что это объект, и создает его.
  • При создании объекта вы указываете список переменных внутри него с помощью синтаксиса: имяСвойства: значениеСвойства.
  • Для присваивания свойству объекта какого-то значения нужно писать не знак =, а ставить после имени свойства двоеточие :.
  • Создать свойство объекта без присваивания ему какого-то значения нельзя.
  • Пары свойство: значение перечисляются через запятую, а не через точку с запятой.
  • Если имя свойства содержит больше одного слова, которые разделен пробелом, то его нужно заключать в кавычки: имя свойства, если оно не содержит пробелов - этого можно не делать.

Пример:

let user = { // создаем объект user
  name: "Влад", // создаем внутри объекта user свойство name со значением "Влад"
  "second name": "Дракула", // если в имени свойства больше одного слова, разделенных пробелом - заключаем имя свойства в кавычки
  age: 400, // в качестве значения свойства может храниться не только строка, но и число
  marriage: false // а также булевое значение или null
};

Важно: если вывести объект в консоль, он будет выглядеть не так, как переменные, хранящие в себе примитивные типы данных (когда выводится значение, хранящееся внутри переменной) - вы увидите всю структуру объекта - все имена свойств и их значения:

console.log(user);

Важно!
Все задачи лучше выполнять в отдельных файлах - index1.html, index2.html и т.д., при необходимости копируя нужный код из одной задачи в другую, и потом меняя его. Для ускорения работы, код можно писать внутри тега "script", а CSS код внутри тега "style".

Работа со свойствами объекта

Воспользуемся предыдущим примером:

let user = { 
  name: "Влад", 
  "second name": "Дракула", 
  age: 400, 
  marriage: false 
};

И добавим после него такой код:

console.log(user.name); // Выведет "Влад"
console.log(user["second name"]); // Выведет "Дракула"
console.log(user.age); // Выведет "400"
console.log(user["marriage"]); // Выведет "false"

Этот код выводит в консоль значение разных свойств объекта.

Из примера видно, что:

  • Если вы хотите получить доступ к свойству объекта (начать работать с со свойством как с отдельной переменной), то это можно сделать, записав ее имя как: имяОбъекта.имяСвойства;.
  • Если имя свойства состоит из больше чем одного слова, разделенных пробелом, то для получения к ней доступа нужно писать так: имяОбъекта["имя Свойства"] (то есть вместо точки после имени объекта вы ставите квадратные скобки, внутри которых пишите имя свойcтва, заключенное в кавычки). Для свойств из одного слова такой способ тоже работает.

Обратите внимание! Чтобы работать с переменной внутри объекта, вы в любом случае сначала должны написать имя объекта, и только после него - имя свойства. По сути user.name - это полное имя переменной.

Поэтому если вы хотите изменить значение, хранящееся внутри свойства объекта, вы пишете: 

имяОбъекта.имяСвойства = "новое значение";

либо

имяОбъекта["имя cвойства"] = "новое значение";

Пример:

user.age = 401; // меняем возраст

user["second name"] = "Цепеш"; // если в имени свойства несколько слов, разделенных пробелом - после имени объекта не ставим точку, а пишем квадратные скобки, внутри которых - имя свойства в кавычках

user.marriage = "под вопросом"; // меняем значение и тип данных, хранящихся в свойстве marriage

console.log(user);

В консоли уже будет объект с другими значения полей age и marriage.

Также можно обратиться к свойству объекта, вместо конкретного имени указав переменную, в которой это имя хранится. Синтаксис: имяОбъекта[имяПеременной]; (так же, как и в случае имени из нескольких слов, только без кавычек).

Пример:

const person = {
  name: "Рокэ",
  "last name": "Алва",
  "eyes color": "сапфировые",
  "hair color": "черные",
  "семейный статус": "НИКОГДА"
}

let marriage = "семейный статус"; // сохраняем в переменной имя нужного нам свойства объекта

console.log(person[marriage]); // из переменной marriage берется ее значение - строка "семейный статус", и подставляется внутрь квадратных скобок. В итоге для JavaScript это выглядит как console.log(person["семейный статус"]);

Задача 7

Создать объект danItStudent, у которого будут такие свойства: имя, фамилия, количество сданных домашних работ. Вывести объект в консоль. После чего спрашивать у пользователя в цикле два вопроса - "Какое свойство вы хотите изменить?", потом - "На какое значение?". Цикл может иметь максимум 3 итерации. Если пользователь нажмет Cancel на любой из вопросов - досрочно прервать цикл. Вывести объект в консоль.

Нажмите, чтобы подсмотреть решение 

const danItStudent = {
  name: "Мирослав",
  "last name": "Полищук",
  homeworks: 30
};

console.log(danItStudent);

for(let i = 0; i < 3; i++) {
  let property = prompt("Какое свойство вы хотите изменить?");
  let newValue = prompt("На какое значение?");

  if(!property || !newValue) {
      break;
  }

  if(property === "homeworks") {
      newValue = +newValue;
  }

  danItStudent[property] = newValue;
}

console.log(danItStudent);

Есть ли у объекта такое свойство?

Разумно определить, если у объекта вообще такое свойство, перед тем как начать с ним работать. Это можно сделать двумя способами:

  1. Знать, что если такого свойства у объекта нет, попытка к нему обратиться вернет undefined.

    Например:

    const person = {
  name: "Рокэ",
  "last name": "Алва",
  "eyes color": "сапфировые",
  "hair color": "черные",
  "семейный статус": "НИКОГДА"
}

console.log(person.wife); // выведет undefined, так как такого свойства у объекта нет

    Следовательно, проведя строгое (через ===) сравнение значения, хранящегося в предполагаемом свойстве объекта с undefined, можно узнать, есть ли вообще у объекта такое свойство.

    Помните, что нельзя создать внутри объекта свойство без присваивания ему значения.

    Пример:

    let person = {
  name: "Рокэ",
  "last name": "Алва",
  "eyes color": "сапфировые",
  "hair color": "черные",
  "семейный статус": "НИКОГДА"
}

if (person.wife === undefined) {
  console.log("У Первого маршала Талига нет жены!");
}

  2. В теории возможна ситуация, когда свойство у объекта есть, но ему сознательно присвоили значение undefined. Значит строгим сравненим дело не решишь. Тут нужно использовать следующую инструкцию:

    "имяСвойства" in имяОбъекта

    Данная конструкция возвращает true, если у объекта вообще есть свойство с таким именем, даже если его значение равно undefined, и false, если его нет.

    Пример:

    const person = {
  name: "Рокэ",
  "last name": "Алва",
  "eyes color": "сапфировые",
  "hair color": "черные",
  "семейный статус": "НИКОГДА",
  "право на трон": undefined,
}

Добавление и удаление свойств объекта

Важно: в любой момент вы можете добавить в объект новые свойства после того, как создали его и инициализировали, просто написав:

имяОбъекта.имяСвойства = значениеСвойства

или

имяОбъекта["имяСвойства"] = значениеСвойства

Пример:

const book = { // создаем объект с название книги и ее автором
  name: "50 оттенков серого",
  author: "Л. Джеймс",
}

console.log(book.name);
console.log(book.author);

book.price = 50; // добавляем объекту стандартную цену
book["akcii price"] = "1 мозг"; // и акционную цену

console.log(book.price); // как видим, значения добавились 
console.log(book["akcii price"]);

То есть, если вы присваиваете (через =) значение несуществующему свойству объекта, оно сначала добавляется к объекту, а потом это свойство инициализируется указанным вами значением.

Перебор свойств объекта

Часто нужно перебрать все свойства объекта. Проблема в том, что их количество в объекте может меняться, как и их названия, особенно если объект создается по результатам того, что ввел пользователь на странице (например - список дел). А значит вы наверняка не можете знать ни сколько будет свойств в объекте, чтобы запустить цикл for с определенными количеством итераций, ни названия этих свойств. И тут на помощь приходит цикл for..in с таким синтаксисом:

for (let ключОбъекта in объект) {
  /* ... делаем что-то с объект[ключОбъекта] ... */
}

Пример:

const musician = {
  name: "Виктор",
  "last name": "Цой",
  status: "Жив!"
};

for(let key in musician) { // сохраняем в переменную key на каждой итерации цикла имя следующего свойства объекта
  console.log(musician[key]); // поскольку в переменной key хранится название свойства объекта, то мы может к нему обратиться через имяОбъекта[имяПеременной]
}

Обратите внимание:

  • Переменную key (тут может быть любое имя) в инструкции называют ключом объекта. Это то же самое, что имя свойства объекта. ключ === имяСвойстваОбъекта.
  • for..in - это цикл, который последовательно перебирает все свойства объекта.
